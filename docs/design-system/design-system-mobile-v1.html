Seção 9 · Checklist

Checklist de Implementação Android

Lista objetiva para transformar este design system em biblioteca Android real.

Tokens & Tema

  • Criar NeoReaderTheme.kt com darkColorScheme completo
  • Adicionar fontes Inter + Playfair Display via Google Fonts Compose
  • Definir Typography com todos os 11 estilos em sp
  • Criar res/values/dimens.xml com spacing tokens em dp
  • Criar res/values/colors.xml com todos os color tokens
  • Definir Shape scale (xs=4dp até xl=24dp)
  • Configurar Elevation tokens (0–5dp)

Componentes base

  • NrButton (primary, secondary, ghost, danger + loading + disabled)
  • NrTextField (outlined, error, disabled)
  • NrSearchBar (M3 search, com filtro)
  • NrBadge (purple, success, warning, error, neutral)
  • NrSwitch, NrCheckbox, NrRadio (48dp touch target)
  • NrTabs (scrollável, indicator 2dp)
  • SkeletonCard + shimmerBrush helper

Componentes de conteúdo

  • BookCard (120dp, cover 2:3, action button, progress bar)
  • ContinueReadingCard (progress card horizontal)
  • BookListItem (72dp, cover, title, meta, action)
  • ContentRow (header + LazyRow com BookCards)
  • EmptyState (ícone + título + desc + CTA)
  • HeroBanner (cover art + overlay + CTA buttons)

Navegação & Overlays

  • NrTopAppBar (home variant + internal variant)
  • NrBottomNavigation (4 destinos + M3 indicator)
  • BookActionsSheet (ModalBottomSheet substituindo dropdown)
  • NrDialog (confirmações, max 340dp)
  • NrSnackbar host configurado no Scaffold root

Acessibilidade

  • Touch target mínimo 48dp em todos os elementos clicáveis
  • contentDescription em todos os ícones e imagens
  • sp para todos os tamanhos de texto (respeita font scaling)
  • Contraste ≥ 4.5:1 para texto (verificar com Accessibility Scanner)
  • Não usar cor como único indicador de status
  • reduceMotion: desabilitar animações se WindowInsets.isReduceMotionEnabled
  • Semântica TalkBack: semantics { } em Cards e itens complexos

Insets & Layout

  • enableEdgeToEdge() no Activity + WindowCompat.setDecorFitsSystemWindows(false)
  • Scaffold com padding de navigationBars + statusBars
  • Bottom sheet com padding de navigationBars (safe area)
  • LazyColumn/LazyRow com contentPadding = PaddingValues(16.dp)
  • Testar em 360dp (mínimo) e 412dp (target)
  • Testar com fontes do sistema em tamanho grande (1.3x, 1.5x)
Anti-patterns a evitar: Não use px hardcoded no código Compose. Não use hover ou onHover. Não crie botões menores que 48dp. Não use Dialog para listas longas — use ModalBottomSheet. Não remova o handle visual do Bottom Sheet (impede que TalkBack identifique como arrastável).
