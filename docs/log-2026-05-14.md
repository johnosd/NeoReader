## 🔴 Bug aberto: Google Sign-In falha no Play Store build

**Sintoma**: tocar em login Google no app instalado via Play Store → loading eterno ou "no credentials available". Picker do Google não abre visualmente.

**No dev build (`npm run android:run`)**: funciona normalmente.

**Logs reveladores**:
```
W Auth: getToken() -> NEED_REMOTE_CONSENT. Service: oauth2:.../auth/lams
W Auth.Api.Credentials: UserRecoverableAuthException: NeedRemoteConsent
I CredentialManager: Remote provider response timed out (3s)
I CredentialManager: Status changed to CANCELED
```

**O que já foi verificado e está correto** (não repita):
- ✅ SHA-1 App signing key (`7f3e44ba...`) registrado no Firebase
- ✅ SHA-1 Upload key (`44b4fbaa...`) registrado no Firebase
- ✅ `google-services.json` baixado pós-registro + substituído + AAB rebuildado
- ✅ Audience "In production" no Google Auth Platform
- ✅ Branding completo (App name, support email, developer contact)
- ✅ Web OAuth Client tem config impecável (JS origins, redirect URI, enabled, no warnings)
- ✅ Device saudável (Gmail/Calendar funcionam normalmente após re-add conta)
- ✅ Conta Google removida e re-adicionada no celular
- ✅ Cache + dados do Play Services limpos, reboot feito

**Hipóteses prováveis** (em ordem):
1. **Anti-abuse do Google na conta `johnscosta2@gmail.com`** — muitas tentativas de auth falhas hoje. Costuma liberar sozinho em 24-48h.
2. **Bug do Credential Manager API v8** + `@capacitor-firebase/authentication` v8 com essa combinação device/account/SHA-1.
3. **Propagação Google Cloud** ainda incompleta.

Adiciona outra conta Google qualquer no celular (Configurações → Contas → Add account). Tenta login no NeoReader com ela.

- ✅ Funcionou → confirma anti-abuse na conta `johnscosta2@gmail.com`. Espera mais 24h ou usa outra conta.
- ❌ Não funcionou → passo 3

### Passo 3 — Testar em emulador Android Studio (15 min)
Cria AVD com Google Play Services. Instala via opt-in link. Tenta login.

- ✅ Funcionou → problema é específico do seu device físico (cache profundo, possivelmente).

Implementa Firebase Auth email+senha como método principal. Google Sign-In vira opcional. App fica 100% funcional sem dependência da config OAuth complexa.
